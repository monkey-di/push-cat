# Правила анализа требований к данным

Эти правила используются агентом data-analyzer при определении структуры инфоблоков.

## Критерий редактируемости

Контент считается редактируемым через админку **только** если в ТЗ явно указано:
- "редактируется в админке"
- "можно менять/задавать в админке"
- "управляется через админку"
- "загружается/добавляется через админку"
- "количество не ограничено" (подразумевает управление в админке)

**Если явного указания нет — контент статический, поле НЕ создаётся.**

## Типы полей

| Данные в ТЗ | type | user_type | property_type в fields |
|-------------|------|-----------|----------------------|
| Заголовок, название, URL | `'S'` | — | `'string'` |
| Описание, текст, контент | `'S'` | `'HTML'` | `'text'` |
| Изображение, фото, документ | `'F'` | — | `'file'` |
| Выбор Да/Нет, категория | `'L'` | — | `'enum'` |
| Привязка к элементам | `'E'` | — | `'element'` |

## Правила именования свойств

Формат: `SECTION_FIELD` в UPPER_SNAKE_CASE

Примеры:
- Секция "Hero", поле "Заголовок" → `HERO_TITLE`
- Секция "About", поле "Описание" → `ABOUT_TEXT`
- Секция "Partners", видимость → `PARTNERS_SHOW`
- Кнопка CTA → `CTA_BTN_TEXT`, `CTA_BTN_LINK`

## Когда создавать отдельный инфоблок

Создавай отдельный инфоблок если:
1. Элементы множественные и количество управляется в админке
2. Элементы имеют сложную структуру (3+ полей)
3. Элементы потенциально переиспользуются на других страницах

Оставляй поля в singleton если:
1. Количество фиксировано и мало (1-3 карточки)
2. Структура простая (заголовок + текст)
3. Используются только на этой странице

## Решения о переиспользовании

Перед созданием нового инфоблока проверь `local/config/iblocks/*.php`:

**Переиспользуй**, если:
- Совпадают 80%+ полей по структуре
- Та же семантика (FAQ = FAQ, Team = Team)
- Уже используется аналогично в другом месте

**Дополни существующий**, если:
- Базовая структура совпадает
- Не хватает 1-2 полей
- Добавление не сломает существующие компоненты

**Создай новый**, если:
- Совпадение < 50% полей
- Другая семантика (FAQ ≠ Преимущества)
- Нужна изоляция данных

## Типичные страничные singleton-структуры

Для типовых секций, вот стандартные поля:

### Hero-блок
```
HERO_TITLE (S) — заголовок
HERO_TEXT (S/HTML) — подзаголовок/описание
HERO_IMAGE (F) — фоновое изображение
HERO_BTN_TEXT (S) — текст кнопки
HERO_BTN_LINK (S) — ссылка кнопки
```

### Info-секция
```
SECTION_TITLE (S) — заголовок
SECTION_TEXT (S/HTML) — текст
SECTION_IMAGE (F) — изображение
```

### CTA-блок
```
CTA_TITLE (S) — заголовок
CTA_TEXT (S/HTML) — описание
CTA_BTN_TEXT (S) — текст кнопки
CTA_BTN_LINK (S) — ссылка
```

### Видимость секции
```
SECTION_SHOW (L: Да/Нет) — показывать/скрывать блок
```
