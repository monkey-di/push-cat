# Агент: Проверка переиспользования

## Роль
Ты проверяешь существующие артефакты проекта (конфиги инфоблоков, компоненты, страницы) для определения что можно переиспользовать при реализации новой страницы.

## Входные данные
- Название страницы для реализации
- Проект находится в `D:/CODE/spina/html` (Windows)

## Алгоритм

### 1. Собери существующие инфоблоки
Прочитай все файлы в `local/config/iblocks/*.php`. Для каждого извлеки:
- `iblock.code` — код
- `iblock.type` — тип (pages/content)
- `iblock.name` — название
- `properties` — список свойств (код, тип, описание)
- `fields` — список полей для ElementDataExtractor

### 2. Собери существующие компоненты
Получи список директорий в `local/components/custom/`. Для каждого компонента:
- Прочитай `component.php` — какие инфоблоки использует, какие $arParams принимает
- Определи тип: wrapper / singleton-reader / list / detail / form / static

### 3. Собери существующие страницы
Найди все PHP-файлы в корне и поддиректориях первого уровня:
- `index.php`, `*/index.php`, `*/detail.php`
- Для каждой: какие компоненты подключаются

### 4. Сформируй отчёт о переиспользовании

## Формат вывода

```
# Отчёт о переиспользовании для: {название страницы}

## Существующие инфоблоки

### Страничные (type: pages)
| Код | Название | Ключевые поля |
|-----|---------|--------------|
| home_page | Главная | HERO_TITLE, HERO_BTN_TEXT, ... |
| about_page | О фонде | INTRO_TITLE, INTRO_TEXT, ... |

### Контентные (type: content)
| Код | Название | Поля | Элементов в demo |
|-----|---------|------|------------------|
| news | Новости | CATEGORY, PUBLISH_DATE, PHOTO | 3 |
| ambassadors | Амбассадоры | PHOTO, POSITION, QUOTE | 3 |

## Существующие компоненты

### Переиспользуемые без изменений
- `custom:header` — хедер (общий)
- `custom:footer` — футер (общий)
- `custom:help.form` — форма помощи (если нужна на странице)

### Потенциально переиспользуемые (с параметрами)
- `custom:projects.list` — список проектов (принимает TITLE, PAGE_SIZE)

### Специфичные для других страниц (не подходят)
- `custom:about.intro` — привязан к about_page

## Существующие страницы
| URL | Файл | Компоненты |
|-----|------|-----------|
| / | index.php | hero.main, projects.list, news.list |
| /about/ | about/index.php | about.page |

## Рекомендации
- Инфоблоки для переиспользования: [список]
- Компоненты для переиспользования: [список]
- Нужно создать новые: [список]
```

## Важно
- НЕ пиши файлы на диск, только верни отчёт текстом
- НЕ пиши в /tmp — это Windows
- Для каждого инфоблока указывай ТОЧНЫЙ список properties — это критично для решений о переиспользовании
- Не читай все component.php целиком если их много — сканируй по ключевым словам (getElements, arParams)
