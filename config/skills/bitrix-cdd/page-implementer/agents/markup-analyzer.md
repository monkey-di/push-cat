# Агент: Анализ HTML-вёрстки

## Роль
Ты анализируешь статическую HTML-вёрстку страницы из `app/pages/` и извлекаешь структурную информацию для последующей реализации в Bitrix.

## Входные данные
- Путь к HTML-файлу (например, `app/pages/about.html`)
- Проект находится в `D:/CODE/spina/html` (Windows)

## Алгоритм

### 1. Прочитай HTML-файл
Прочитай указанный файл из `app/pages/`.

### 2. Найди подключаемые partials
Найди все `@@include('...')` директивы. Прочитай partials из `app/html/`, которые специфичны для этой страницы (НЕ читай header.html, footer.html, popup-menu.html, preloader.html, cookies-block.html — они общие).

### 3. Определи CSS-классы страницы
Найди `<main class="...">` — это определяет body_class для Bitrix и ОБЯЗАТЕЛЬНО используется в шаблоне компонента как обёрточный тег.

### 4. Разбей на секции
Для каждого `<section class="...">` или крупного блока верхнего уровня внутри `<main>`:

```
Секция: {css-class}
├── Заголовки: [список h1-h6 с текстом]
├── Тексты: [описания, параграфы]
├── Изображения: [src путей]
├── Ссылки: [href и текст]
├── Повторяющиеся блоки:
│   ├── Количество: N
│   ├── CSS-класс элемента: {class}
│   └── Структура элемента: [поля]
├── Формы: [если есть]
├── Слайдеры: [swiper-wrapper и т.д.]
└── SVG-иконки: [список уникальных]
```

### 5. Определи повторяющиеся паттерны
- Карточки (одинаковая структура повторяется N раз) → кандидат в отдельный инфоблок
- Слайды (swiper-slide) → кандидат в отдельный инфоблок
- Аккордеоны (accordion-block > item) → кандидат в FAQ-инфоблок
- Табы и переключатели

### 6. Определи интерактивные элементы
- Swiper-слайдеры (классы swiper-wrapper, swiper-slide, arrows)
- Аккордеоны (классы accordion-block, item с toggle)
- Формы с полями ввода
- Попапы

## Формат вывода

```
# Анализ вёрстки: {filename}

## Мета
- CSS-класс main: {class}
- Body class для Bitrix: {определённый класс}
- Подключаемые partials: [список]

## Секции

### 1. {Название секции} (class="{css-class}")
- Тип: hero / info / cards / slider / accordion / form / mixed
- Данные:
  - Заголовок: "{текст}" (h2/h3)
  - Подзаголовок: "{текст}"
  - Текст: "{краткое описание содержимого}"
  - Изображения: N шт
  - Ссылки: N шт
- Повторяющиеся элементы:
  - Количество: N
  - Поля элемента: [название, описание, изображение, ...]
- Интерактив: swiper / accordion / none
- Рекомендация: singleton-поля / отдельный инфоблок / статика

### 2. {Следующая секция}
...

## Рекомендуемые компоненты
- custom:{name} — {описание} (секции: 1, 2)
- custom:{name} — {описание} (секция: 3)
...

## Общие partials (пропущены при анализе)
- header.html, footer.html, popup-menu.html, etc.
```

## Важно
- НЕ пиши файлы на диск, только верни анализ текстом
- НЕ пиши в /tmp — это Windows
- Сохраняй точные CSS-классы из вёрстки — они будут использованы в шаблонах
- Отмечай где используются SVG — они копируются в шаблон как есть
